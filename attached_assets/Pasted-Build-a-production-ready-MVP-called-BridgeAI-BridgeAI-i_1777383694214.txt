Build a production-ready MVP called BridgeAI.

BridgeAI is a standalone plug-and-play AI-to-AI integration tool. It lets users connect different AI providers, assign a project, and let the connected AIs communicate in one shared conversation, divide tasks, add input when needed, and collaborate autonomously.

The product is NOT a chatbot platform and NOT a project management tool. It is an integration bridge between AI systems.

Core product promise:
“Plug in your AIs. Give them a project. BridgeAI lets them work together.”

Build this as a full-stack web app.

Use:
- Next.js
- TypeScript
- Tailwind CSS
- Node API routes
- SQLite for local MVP storage
- Prisma ORM
- OpenAI API integration as the first working provider
- Mock adapters for Claude, Manus, Replit, Gemini until real API keys are added
- Clean modular architecture so real providers can be added later

Main user flow:
1. User opens BridgeAI.
2. User creates a new collaboration session.
3. User enters a project goal.
4. User selects which AI agents to include.
5. User assigns roles or lets BridgeAI auto-assign roles.
6. BridgeAI creates a task plan.
7. Agents communicate in one shared conversation.
8. Each agent adds input based on its role.
9. BridgeAI routes tasks to the best-fit agent.
10. BridgeAI shows task status, agent messages, costs estimate, and final output.
11. User can approve final action.

Core backend features:

1. Agent adapter interface

Create a reusable TypeScript interface:

interface AgentAdapter {
  id: string;
  name: string;
  provider: string;
  capabilities: string[];
  runTask(input: AgentTaskInput): Promise<AgentTaskResult>;
}

2. Implement OpenAI adapter

Use OpenAI Responses API.
Model should default to a low-cost model like gpt-4.1-mini or current equivalent configurable through env.
The OpenAI adapter should accept:
- system role
- project goal
- shared memory summary
- task instruction
- previous messages

Return:
- message text
- suggested next tasks
- completion status
- confidence

3. Implement mock adapters

For Claude, Manus, Replit, Gemini, and Perplexity, create mock adapters that generate realistic placeholder responses without API calls.

Each mock adapter should behave based on role:
- Manus mock: research/execution style output
- Replit mock: builder/coding/deployment style output
- Claude mock: reviewer/writing/logic critique
- Gemini mock: multimodal/contextual analysis
- Perplexity mock: research summary

This lets the product work immediately without needing all real APIs.

4. Task router

5. Cost-aware routing

Implement a simple cost-aware router.

Rules:
- Use mock/free agents when available for low-risk tasks
- Use OpenAI for reasoning, planning, creative direction, and final review
- Use Replit adapter for build tasks
- Use Manus or Perplexity adapter for research
- Use Claude adapter for review if selected
- Add estimated cost per task
- Show total estimated cost in UI

6. Shared memory

For each session maintain:
- project goal
- selected agents
- autonomy mode
- current summary
- decisions made
- tasks
- messages
- final output

After each agent response, update shared memory summary.

7. Agent conversation loop

Create a controlled multi-agent loop:
- Step 1: BridgeAI creates project plan
- Step 2: Router assigns tasks
- Step 3: Assigned agent responds
- Step 4: BridgeAI updates memory
- Step 5: Reviewer agent reviews if needed
- Step 6: Builder agent revises if needed
- Step 7: Final QA produces final result
- Step 8: User approval required before deployment/publishing

Prevent infinite loops:
- max 8 agent turns per “run full supervised workflow”
- max 2 retries per task
- stop if final QA says complete
- stop if approval required
